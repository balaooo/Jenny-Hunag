I am the sole designer for this project and have been working closely with PM and engineers throughout the development process.

The objective of this project is to create an intuitive trust inquiry system, enabling customers to easily access information and replace paper-based processes.

＊This website was originally written in Mandarin and has been translated into English for the purpose of this portfolio.

Background

Although the current system already has a trust inquiry feature, it lacks a personal trust inquiry function. Therefore, in this version update, we have planned to include a personal trust inquiry service on the website to enhance the overall trust inquiry functionality.

*Personal Trust Inquiry: This feature allows trust owners to inquire about the beneficiaries, inventory, transaction details, and other related information of their trusts.

Problems

  • 1. How to deal with multiple Login Paths?
    The existing personal trust system consists of two different types of trusts, each with its own authentication process. Users only need to authenticate themselves for the specific items they want to inquire about. Therefore, each user will have their own login path based on their authentication requirements.
  • 2. How to present the query options?
    Product team proposes providing a wide range of query options. For a single trust, users can query four different sub-items: basic information, inventory details, transaction history, and statement download. To avoid a cluttered interface, careful planning is needed to organize these options effectively

  • .

  • How to express proprietary terms?

    • In the banking industry, there are specific terms used for different types of trusts, such as "self-beneficiary trust" and "third-party beneficiary trust." The authentication process for these two types of trusts varies. it is important to address how to present these proprietary terms in a way that is easily understandable to users.

    User Interface

    Based on pain points and user characteristics, I include 4 key elements in UI/UX design:

    1. Main visual that clearly presents the activity flow
    2. Use icons to assist in explaining complex banking terms
    3. Highlight various explanatory sections with color blocks
    4. Manage lengthy explanation texts with pop-ups or URLs
    1. Main visual that clearly presents the activity flow

    The main visual utilizes a flowchart with color blocks representing different steps to provide a concise overview.

    • Well-acquainted customers can immediately proceed with the activity registration by using the CTA button below.
    • If new customers want to learn more about the activity details, they can refer to the color blocks and find corresponding detailed explanations in the form below.

    The original design: Directly leads to a text explanation section, which contains a large amount of text. This can create a burden for users to read and may lead them to give up participating in the activity.

    The new design: Charts and color blocks make the activity content lively, enhancing understanding and absorption of information.

    2. Use icons to assist in explaining complex banking terms
    • Ensure that the icon prompts are represented with simple and easily understandable graphics.
    • Adhere to the design system of the bank for icon usage (e.g., voucher, points icon) to avoid confusion across different activities.
    • Maintain consistency by using the same set of icons in all promotional materials associated with the activity (e.g., EDMs, posters, collateral).
    3. Highlight various explanatory sections with color blocks

    To prevent user confusion when navigating through the activity's various sections, utilize different color blocks to represent different functions. This will help users easily identify and distinguish between different sections of the page, especially when using the fixed navigation for quick jumps between sections.

    4. Manage lengthy explanation texts with pop-ups or URLs

    Complex task steps are accompanied by explanatory text and images to assist users.

    • Users don't need to search for the steps of these activity tasks on their own.
    • To maintain a clean interface, only icons are used on the main screen, while the detailed content is displayed in pop-up windows or linked to separate explanatory websites.
    • Mobile users can also read information easily in this format.

    The original design: Displayed all the information on the screen, which made it appear cluttered and difficult to read.

    The new design: Incorporates icons as buttons, allowing customers to click on the tutorials they need, making the interface cleaner and more streamlined.

    Takeaways

    Outcome:
    • By promoting the event on various social media platforms and driving traffic to this webpage, the number of participants exceeded expectations by approximately 10%. As a result, the team decided to extend the event until the end of the year, with the goal of achieving a participation rate that is 15-20% higher than initially anticipated.
    • Due to the successful online promotion, starting from H2, I will also be responsible for designing offline promotional materials for the event, such as posters and standees in branches.
    Optimization:
    • To further enhance the user experience, we will provide additional guidance by adding "i" icons next to each task in the introductory flowchart. These icons will allow users to quickly navigate to the tutorial section below.
    • In anticipation of the monthly announcement of the winners for this event, we are planning to implement a tab-based navigation system to prevent the webpage from becoming excessively long. This will ensure easy access for users while maintaining a clean and organized layout.
